Sample with a purpose!

When you're trying to obtain knowledge from a data source, such as finding light sources, you can't rely on luck to get accurate estimations. We see that uniform hemisphere sampling is essentially throwing darts while blindfolded. It will eventually converge to the correct light sources, but we'll hit several non-light objects in the process. This gives us noisy renderings of scenes. On the other hand, importance sampling specifically samples directions towards light sources. Each sample will most likely contribute to the final rendering. The resulting scene will be much smoother, and we can finally render scenes with point lights.

Part 2: Bounding Volume Hierarchy

Walk through your BVH construction algorithm. Explain the heuristic you chose for picking the splitting point.

Our BVH construction algorithm constructs a binary tree where each node has a bounding box, which we expanded as necessary. If the current node has less than max_leaf_size primitives, then it's a leaf node. We give it iterators to the start and end of its primitives, and return. Afterwards, we handle the internal node case. We select the splitting axis in a way that creates a balanced tree. Our splitting heuristic is a median split. First, we determine which axis has the largest range in the bounding box. Then, we sort the primitives based on their bounding box's centroid along the chosen axis. We split the primitives in the middle, and finally recursively call construct_bvh() on both sides of this half. This reduces the expected runtime from O(n) to O(log(n)), significantly speeding up the rendering process.

Part 3: Direct Illumination

Walk through both implementations of the direct lighting function.

For hemisphere sampling, we iterate through each sample in num_samples. For each sample, we randomly grab a direction uniformly across the hemisphere (w_in) with hemisphereSampler->get_sample(). We create a new ray from hit_point (+ EPS_D to avoid self-intersection) in w_in's direction converted to world space, and see if it intersects an object in the scene. If the intersected object in question is a light source, we accumulate the contribution according to the BSDF formula described in lecture, where the PDF is 1 / (2 * PI). Finally, we normalize L_out by the number of samples, which gives us the Monte Carlo estimate.

For importance sampling, we iterate through each light in the scene. First, we define num_samples as 1 if it's a delta light, and ns_area_light for area lights. For each sample, we obtain w_in_world as the incoming direction with sample_L. We also get the distance to the light, and the light's PDF. We calculate the cosine between the surface normal and the light's direction. We use that to cast a shadow ray from hit_point to the light source. We add the light's contribution to total_light if it is in front of the surface (cosine test) and if there's nothing blocking hit_point and the light source. We then accumulate L_out based on the BSDF as described in lecture. Finally, we average all the samples for each light, and take this sum as the total direct lighting.

Part 4: Global Illumination

Walk through your implementation of the indirect lighting function.

For the indirect lighting function, we first check if the ray has exceeded its maximum depth. If so, we only return the one bounce radiance. If it hasn't exceeded the max depth, we use importance sampling with isect.bdsf->sample_f() to sample the incoming direction (w_in). This also lets us get the PDF. Then, we defined a variable called termination_prob for Russian roulette that's set to 0.3. While Russian roulette isn't stopping our path, we convert the sampled direction to world space. We create a new ray with an increased depth, and check for intersections. If there is an intersection, we recursively call at_least_one_bounce_radiance() and weigh the contribution by the BSDF value, cosine term, and PDF. Finally, after we trace every path and accumulate their contributions, we return the summed L_out.

Part 5: Adaptive Sampling

Explain adaptive sampling. Walk through your implementation of the adaptive sampling.

Adaptive sampling is essentially a sampling technique where you adjust the number of samples based on how complex a pixel's lighting is (i.e. high variance). We implemented this with s1 (illuminance sum), s2 (squared illuminance sum), and n (number of samples) as described in the writeup. We have a for loop that iterates over samples until we reach the max amount of samples, or we reach the convergence condition. We calculate x_k with radiance.illum() and use it to update the values for s1 and s2. Then, as suggested in the writeup, we check convergence every samplesPerBatch=32 pixels. We calculate μ and σ² properly, then use them to see if I is less than maxTolerance * μ. If so, the sample has converged, and we can break out of the for loop. We then divide the radiance sum by n to obtain the average radiance, update the sample buffer, and update the sample count buffer accordingly.
